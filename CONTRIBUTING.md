# Contributing to Sentinel GRC

First off, thank you for considering contributing to Sentinel GRC! It's people like you that make the open-source cybersecurity community so powerful.

## How Can I Contribute?

### 1. Expanding Regulatory Mappings
The core of Sentinel GRC is its Explainable AI mapping. You can contribute by adding new frameworks (e.g., GDPR, DORA, HIPAA) to our matrix.
* Open config/regulatory_matrix.json
* Add your mapping following the existing schema.
* Submit a Pull Request!

### 2. Adding Support for New IaC Tools
Currently, we support Terraform and basic supply chain analysis. If you want to build parsers for Kubernetes manifests, AWS CloudFormation, or Ansible, your PRs are highly welcome.

## Pull Request Process
1. Fork the repo and create your branch from main.
2. If you've added code that should be tested, add tests.
3. Update the README.md with details of changes to the interface or new features.
4. Ensure your PR description clearly describes the problem and solution.

## Development Setup
Please refer to the **Installation & Setup** section in the README.md to get your local environment running with Ollama and Llama 3.

## Code of Conduct
By participating in this project, you agree to maintain a respectful and inclusive environment for everyone.
